Step 3 - Play

Once you are in a game there are basically 2 game activities:
  1. Check into Venues via Four-square and Swarm - your checkins will trigger game events. See Four-square activity below
  2. Use the four-stragerey interface to move Game units between game venues to gain advantage, fight battles, and score points

Games have a specified start and end date.

Foursquare Activity

  • When you checkin on Foursquare you will generate game activity for all active games you are a player in
  • You are limited to 5 game checkins per game per day. The game is run on US Eastern time so if you checkin 5 times in one day any subsequent checkins will not generate additional game activitys until midnight Eastern time when the count is reset for the next day.
  • Game checkins will generate one of the following game activities
    • If the venue a player checks into is unoccupied --- Then a new unit for the checking in player will be generated in that location. The unit will be named after the venue name. The initial troop strength will be based on the size of the venue (based on foursquare stats)
    • If the venue is already occupied by the player that is checking in -- Then that unit's troops will be increased. The new troop count is determined by the same rules as when the venue is not occupied
    • If the venue is already occupied by an opponent -- Then the checking in player receives no additional troops BUT they will receive an intelligence report which will provide information on the unit at that location

Moving Units

  • Units are moved by accessing the Game activity page for the game in question, this is done by selecting the game from the 'player home' page
  • To move a unit
    • Find the row for the desired unit in the 'Your units' list
    • Click the column in that row for the venue you wish to move to
    • The game will calculate the transit time required and ask you to confirm
    • If you confirm the unit will start to move
  • Once a unit is moving you cannot stop, re-route, or redirect it. It is gonna go there.
  • All units travel at 50 miles per hour
  • Units travel in straight lines, the most direct route, regardless of geography, roads, lakes, mountains, oceans, borders, etc
  • Units in transit cannot interact with any other units
  • Units in transit are not considered when scores are tabulated
  • If a unit arrives at a venue where that player has another unit then those units merge, the troops are added together and the new unit takes the name of the larger of the two merged units. --- UNITS CANNOT BE UN-MERGED or split in any way.
  • If a unit arrives at a venue occupied by a unit belonging to another player, THEN a battle will ensue......

BATTLES

If a unit arrives at a venue that is occupied a battle will ensue. The rules for battles are as follows:
  • Battles end when one side or the other is annihilated - there are no tactical withdraws - more like thunderdome, one man leaves.
  • The battle process follows the classic board game 'risk' combat system
    • The Attacker gets one six sided di per 'troop' with a max of 3
    • The Defender gets one six sided di per 'troop' with a max of 2
    • The dice are thrown and the top di for each side are paired for comparison.
    • The best number of the attacker is paired against the best di for the defender.
    • If both sides have at least 2 dice then their 2nd best values are likewise paired to each other.
    • The looser of each pairing of di looses a troop
    • If a pairing is a tie then the defender wins
    • This process is repeated until one side is ANNIHILATED
    The winner (what is left of them) will occupy the venue The looser's only consolation prize is that they get an accurate intelligence report on the unit that defeated them

Scoring

The scoring system is as follows:
  • Scoring happens once per day at midnight US Eastern time
  • Only Garrioned units participate in scoring, units in transit do not count
  • Unit size is irrelevent when it comes to scoring
  • Scoring works as follows
    • All occupied venues are sorted by size (based on forsquare stats)
    • Each venue provides it's occupier with points based on it's size relative to the others
    • The occupier of the smallest venue gets 1 point for that venue
    • The occupier of the second smallest venue gets 2 points for that venue
    • The occupier of the third smallest venue gets 3 points for that venue
    • and so on...

Winning the Game

Simple - player with the highest score when the game end date hits is the winner.
